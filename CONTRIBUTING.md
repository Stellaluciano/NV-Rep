# Contributing to NV-Rep

Thank you for your interest in contributing to NV-Rep.

## Ways to Contribute

- Report bugs and edge cases through GitHub Issues.
- Propose method extensions and benchmark tasks.
- Improve documentation, tutorials, and reproducibility assets.
- Contribute tests, configuration templates, and experiment tracking utilities.

## Development Workflow

1. Fork the repository and create a feature branch.
2. Keep pull requests focused and well-documented.
3. Add or update documentation for behavioral changes.
4. Ensure all checks pass before requesting review.

## Pull Request Checklist

- [ ] Linked issue or motivation statement.
- [ ] Clear summary of changes.
- [ ] Documentation updated (if applicable).
- [ ] Reproducibility notes added for experiments.

## Code Style and Quality

- Follow consistent style for Python/YAML/Markdown files.
- Prefer explicit configuration over hardcoded experiment settings.
- Keep research code modular and interpretable.

## Community Standards

By participating, you agree to abide by the [Code of Conduct](CODE_OF_CONDUCT.md).
